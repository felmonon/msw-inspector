# Code of Conduct

## Purpose

`msw-inspector` should be a practical, low-friction place to discuss mock coverage, API testing, bugs, and improvements. This code of conduct applies to project spaces, including issues, pull requests, discussions, and maintainer communication related to the project.

## Expected behavior

- Be respectful and direct.
- Assume good intent while still asking for clarification when something is unclear.
- Keep feedback specific to the technical work and its impact.
- Prefer reproducible examples, links, and concrete suggestions.
- Make room for contributors with different backgrounds, experience levels, and communication styles.

## Unacceptable behavior

- Harassment, threats, insults, or sustained personal attacks.
- Discriminatory language or behavior.
- Publishing private information without explicit permission.
- Sexualized language or imagery in project spaces.
- Spam, trolling, or intentionally derailing technical discussion.

## Enforcement

Maintainers may edit, hide, or delete comments; close issues or pull requests; restrict participation; or block accounts when behavior harms the project or its contributors.

If you need to report a conduct concern, contact the maintainer through GitHub with enough context to understand what happened. Maintainers will handle reports as privately as practical and will prioritize the safety of affected contributors.
